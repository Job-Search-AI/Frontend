# 문제 요약
최종 AI 응답 텍스트에서 백엔드가 전달한 줄바꿈(`\n`)이 화면에서 줄바꿈으로 표시되지 않고 한 문단으로 보이는 현상.

# 영향 범위(서비스/사용자/기능)
- 서비스: 검색 결과 요약 가독성 저하.
- 사용자: 응답 구조(문단/항목) 파악 어려움.
- 기능: `AI 응답 결과` 카드의 본문 텍스트 렌더링.

# 재현 조건
- 최종 응답(`user_response` 또는 `message`)에 실제 개행(`\n`) 또는 이스케이프 문자열(`\\n`)이 포함됨.
- 프론트에서 기본 white-space 렌더링(`normal`) 또는 `\\n` 미정규화 상태일 때.

# 가설 목록
1. 렌더링 엘리먼트가 기본 `white-space: normal`이라 실제 개행이 시각적으로 반영되지 않는다.
2. 백엔드 응답이 실제 개행이 아니라 문자 `\\n` 형태로 전달되어 줄바꿈이 일어나지 않는다.
3. API 프록시 경유 시 문자열이 이중 이스케이프될 가능성이 있다.

# 실패 원인 분석
단일 원인이 아니라 두 케이스가 함께 존재할 수 있음.
- 실제 개행 문자열 케이스: white-space 설정 부재로 화면 줄바꿈 미반영.
- 이스케이프 문자열 케이스: `\\n`을 개행으로 변환하는 정규화 로직 부재.

# 의사결정 근거
- 사용자 입력/백엔드 구현 차이로 실제 개행/이스케이프 개행이 혼재할 수 있어 프론트에서 방어적으로 처리하는 것이 안전.
- `whitespace-pre-line`은 과도한 공백 보존 없이 줄바꿈만 반영해 UI 영향이 작음.
- 정규화는 응답 본문 렌더링 범위로 한정해 부작용을 최소화.
